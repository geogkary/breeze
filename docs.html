Configuration

Every theme requires a Config.php and a Theme.php file.

Webmart can collect and handle information based on your theme's Config.php file, like auto-loading bootstrap, fonts and which GET/POST data to accept.

Some of the information is stored into Variables and some are passed into Templates. Of course, you can basically do pretty much anything else you like.

Required (see Routing):

                                        class Config
{
    public static $routes = array();
    public static $redirects = array();
}

Supported:

                                        $version = ''; /** your theme's version (see Templates) */
$cookies = array(); /** cookies to accept and collect */
$query = array(); /** GET data to accept and collect */
$data = array(); /** POST data to accept and collect */
$noindex = array(); /** pages to avoid in robots.txt */
$libs = array(); /** Google libraries to auto-load */
$fonts = array(); /** Google fonts to auto-load */
$bootstap = 'bundle' || true || false; /** load bootstrap */
$hotjar = true || false; /** activate the Hotjar snippet */
$gtag = ''; /** activate Google Analytics */

Feel free to use the boilerplate configuration file.

Routing

Webmart is built on top of mikecao/flight, which means everything is routed automatically.

Routes

Create a $routes array inside your theme's Config class to define your sitemap. Your homepage is routed automatically, so no need to define it with an empty string.

Here's a simple static sitemap:

                                            public static $routes = array(
    'about',
    'about/team', /** use / to define subpages */
    'contact',
    'how-it-works'
);

You can also go fully dynamic by leveraging arguments:

                                            public static $routes = array(
    'about',
    'about-(subpage)'
    'articles',
     'articles-(year-(month-(name)))'
);

See Controllers below on how to use dynamic routing.

Redirects

Create a $redirects array inside your theme's Config class to define your redirects, grouped by an HTTP response. Make sure your redirect source starts with an /.

For example, you might want to permanently move about-us to the about page: 

                                            public static $redirects = array(
    '301' => array(
        '/about-us' => 'about/' /** redirect string starts with an / */
    )
);

You can also redirect manually with Webmart::redirect().

404

Requests which are not configured as either routes or redirects trigger a 404 response and override the template to 404.

Constants

Webmart offers some ready-to-go constants to help you navigate easily inside the core installation and your selected theme.

                                        /** Core */

WM_ROOT /** Your working directory */
WM_DIR /** Webmart's directory */

/** Webmart */

WM_DIR_ENGINE /** webmart/engine/ */
WM_DIR_CORE /** webmart/engine/core/ */
WM_DIR_LIBS /** webmart/engine/libs/ */
WM_DIR_THEMES /** webmart/themes/ */

/** Theme */

WM_DIR_THEME /** webmart/themes/{THEMEFOLDER}/ */
WM_DIR_ASSETS /** webmart/themes/{THEMEFOLDER}/assets/ */
WM_DIR_CLASSES /** webmart/themes/{THEMEFOLDER}/classes/ */
WM_DIR_CONTROLLERS /** webmart/themes/{THEMEFOLDER}/controllers/ */
WM_DIR_JSON /** webmart/themes/{THEMEFOLDER}/json/ */
WM_DIR_TEMPLATES /** webmart/themes/{THEMEFOLDER}/templates/ */

The installation wizard generates a wm.php file and defines the following constants:

                                        WM_THEME /** name of your selected theme's folder */
WM_FOLDER /** name of your working subdirectory */
WM_URL /** your base URL (ex. http://mywebsite.com/) */
WM_ROBOTS /** generates a robots.txt file based on your theme's settings */
WM_DEBUG /** enables debugging and loads your theme's functions.php file */

To restart the wizard, simply delete the file.

Variables

Webmart pre-packs important data, like the current URL, assigned page, cookie data and more into a collection of variables. Use Webmart::get() to retrieve the data.

                                        'url' /** ex. mywebsite.com/about/team */
'cookies' /** retrieved cookies allowed by theme */
'query' /** retrieved GET data allowed by theme */
'data' /** retrieved POST data allowed by theme */
'page' /** based on last call (ex. about/team/ => team) */
'template' /** based on first call (ex. about/team/ => about) */

You can override these settings in your theme with Webmart::set().

Methods

Webmart provides some handy methods to help you take care of repetitive tasks, like loading assets and passing and retrieving data across your theme.

Available methods:

  • set, store a variable
  • get, retrieve a variable's value
  • pass, pass a variable into your templates
  • asset, add a CSS/JS file
  • style, append inline CSS
  • script, append inline JS
  • font, add a Google font
  • library, add a Google library
  • bootstrap, load Bootstrap
  • form, create a form with a closure
  • redirect, redirect to another page

Webmart::set($name, $value)

Create a variable and store its value:

                                            Webmart::set('myvalue', 'Hello world.');

Webmart::get($name)

Retrieve the value of a variable:

                                            $myvalue = Webmart::get('myvalue' || null);

If no name is provided, all set variables are returned as an array.

Webmart::pass($name, $value)

Pass a value into your templates:

                                            Webmart::pass('myvalue', $myvalue); /** in your controller */
echo $myvalue; /** in your template */

Webmart::asset($type, $src, $async)

Add an internal or external CSS/JS file:

                                            $async = true || false;
Webmart::asset('js', 'vue.min', $async);

By default, Webmart looks for the asset inside your theme's assets directory. If the second argument is an external link, Webmart will just append it as is.

Use the $assets variable in your Templates.

CSS/JS files matching the current page (ex. about.css) are auto-included.

Webmart::style($css)

Append inline CSS:

                                            Webmart:style(array(
    'body' => array(
        'background' => "#fff"
    )
));

Auto-passed into the $assets variable in your templates.

Webmart::script($js, $async)

Append inline JavaScript:

                                            $js = 'console.log(1)';
Webmart::script($js, true || false);

Auto-passed into the $assets variable in your templates.

Webmart::font($name, $config)

Load a Google Font:

                                            Webmart::font('Fira Sans', array(
    'weights' => array(
        '300',
        '400'
    ),
    'subsets' => array(
        'latin-extended'
    )
));

Auto-passed into the $assets variable in your templates.

Webmart::library($name, $config)

Load a supported Google library:

                                            /** 2.0.0 */
Webmart::library('jquery', '4.3.1');
Webmart::library('maps', '{API_KEY}');

Auto-passed into the $assets variable in your templates.

Webmart::bootstrap($bundle)

Load the latest version of Bootstrap:

                                            Webmart::bootstrap(true || 'bundle');

Auto-passed into the $assets variable in your templates.

Webmart::form($config, $fields, $closure)

Generate a PHP form with a closure:

                                            $config = array(
    'name' => '', /** name of template variable */
    'method' => 'GET/POST', /** config what your theme allows first */
    'submit' => '' /* name of submit button and request */
);

$fields = array(
    'fieldname1' => array(
        'type' => 'text/textarea/email/password/number/phone',
        'heading' => '',
        'label' => '',
        'placeholder' => '',
        'class' => ''
    ),
    'fieldname2' => array(
        'type' => 'select/radio/checkbox',
        'heading' => '',
        'label' => '',
        'options' => array(
            'option1',
            'option2'
        ),
        'class' => ''
    )
);

Webmart::font($config, $fields, function($response) {
    return $response;
});

Webmart performs basic checks and displays relative messages in the $response.

In your closure, you can always return the new array with the response to override functionality and/or display your custom errors and messages.

Webmart::redirect($page, $status)

Redirect to another page:

                                            Webmart::redirect('welcome/', '200' || null);

By default the status code is 303.

Controllers

Webmart allows you to build your themes as full-fledged web applications and websites with a fully supported MVC logic.

Every theme requires a Theme.php controller at the root of your theme's folder.

The Theme class acts as your global controller, so you can always extend it.

Pages

You can create additional controllers inside your theme's {MYTHEME}/controllers/ subdirectory to handle calls to internal pages. Mind you, the names are case sensitive.

Controllers are loaded automatically in sequence, based on the request.

For example, let's say you have routed about/team/. Webmart will look for the About controller and then for the AboutTeam controller, when the request for that page is made.

Webmart creates an instance with the last controller found.

                                            /** go in sequence */
class About extends Theme {}
class AboutTeam extends About {}

/* or not */
class AboutTeam extends Theme {}

new AboutTeam() /** executed by Webmart */

If there are no sub-controllers, a new Theme() instance is created.

Instead of controllers, you can also use class methods:

                                            class About extends Theme
{
    public function routeTeam() {} /** always route{PAGE} */
}

$controller = new About(); /** both executed by Webmart */
$controller->routeTeam();

Webmart looks for class methods that match the current page, so you can just altogether control your pages inside the Theme controller, instead of creating sub-controllers.

2.0.1 Pages with dashes are changed to underscores (ex. how-it-works):

                                            class How_it_works {}
routeHow_it_works();

Your homepage is always named Home.

404 requests are only handled by the class method route404().

Arguments

You can always use dynamic arguments in your Routing, pretty much in the same MVC logic.

Let's say you've routed blog-(year-(month-(name))) which might translate to a blog/2019/05/hello-world/ request.

In this case, Webmart doesn't look for sub-controllers or class methods for the subpages, but instead provides you with those arguments directly:

                                            /** using the controller */
class Blog extends Theme
{
    public function __construct($routes) {
        $routes[0] = '2019';
        $routes[1] = '05';
        $routes[2] = 'hello-world';
    }
}

/** using a class method */
class Theme
{
    public function routeBlog($routes) {}
}

Routing is disabled, so you need to manually trigger a 404 response (ex. article doesn't exist).

Templates

Webmart autoloads your templates with a basic template engineering logic.

In sequence, it renders the header.php template, then your set template and finally the footer.php template.

Your templates need to be inside the templates/ subdirectory and need to be in lowercase. For the Controllers examples, the final rendering of the templates are:

                                            about.php
team.php /** for an about/team/ request or just team/ */
how_it_works.php /** ex. how-it-works/ */
blog.php /** blog/.../ still loads blog.php */

You can always override the template with Webmart::set().

Any variables you have passed with Webmart::pass() can be directly called in any of the templates. Webmart also prepacks the following for you:

                                            echo $assets;
echo $head;
echo $body;

Your homepage is rendered through the home.php template.

404 requests are rendered with the 404.php template.

More

You can pretty much do anything you like.

Themes are treated as standalone web applications and websites. Webmart just provides the basics for you, like Constants, Variables, Methods and Routing.

Check out the GitHub organisation for free themes.

Libraries

Webmart supports loading external PHP libraries. To use your libraries in an easier way, upload them to the webmart/engine/libs/ subdirectory and use WM_DIR_LIBS to call them.

Webmart will auto-support a few libraries in future versions.
